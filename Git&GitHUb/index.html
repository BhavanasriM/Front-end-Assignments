Git is a powerful version control system with a wide array of commands. Here are some of the most commonly used Git commands, categorized by their function:

Setting up a Repository:

  • git init: Initializes a new Git repository in the current directory.
  • git clone : Clones an existing remote repository to your local machine.

Saving Changes:

  • git add : Stages a specific file for the next commit.
  • git add .: Stages all changes in the current directory for the next commit.
  • git commit -m "Commit message": Records the staged changes to the repository with a descriptive message.
  • git status: Shows the status of your working directory and staged files, indicating untracked, modified, and staged changes.

Branching and Merging:

  • git branch: Lists all local branches.
  • git branch : Creates a new local branch.
  • git checkout : Switches to a different branch.
  • git checkout -b : Creates a new branch and switches to it.
  • git merge : Merges the specified branch's history into the current branch.

Working with Remote Repositories:

  • git remote add : Connects your local repository to a remote repository.
  • git fetch: Downloads objects and refs from another repository.
  • git pull: Fetches changes from a remote repository and merges them into your current branch.
  • git push : Uploads local branch commits to the remote repository.

Inspecting History:

  • git log: Displays the commit history for the current branch.
  • git diff: Shows the differences between files or commits.

Undoing Changes:

  • git reset --hard : Resets the current branch to the specified commit, discarding all changes.
  • git reset --soft : Resets the current branch to the specified commit, keeping changes in the staging area.
